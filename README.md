## What is LocalSkill?

In India, millions of skilled local workers — electricians, plumbers, tailors, tutors — have no digital presence. People who need these services rely entirely on word of mouth with no easy way to find or contact trusted local help.

**LocalSkill** solves this by giving providers a simple platform to list their services, and letting users discover, search, and book them directly — no middleman, no commission.

---

## Features

### For Users
- 🔐 Signup and login with role-based access
- 🔍 Real-time search — filter by title, role, or location instantly
- 📋 Browse all available local service listings
- 📩 Book any service with name, phone, and a message
- 📞 One-tap **Call / WhatsApp** button on every listing

### For Service Providers
- ➕ Post a new service listing with title, role, location, contact
- ✏️ Edit existing listings anytime
- 🗑️ Delete listings
- 👁️ Switch between provider and user view

### Platform
- 🔒 **Role-based UI** — providers and users see different controls
- 📦 All bookings stored in database with auto-timestamp
- ⚡ **Interactive API docs** auto-generated at `/docs`
- 🌐 CORS configured for frontend-backend communication
- 🎨 Production-grade UI with smooth animations and clean typography

## Data Models

### User
| Field | Type | Description |
|---|---|---|
| id | Integer | Primary key |
| name | String | Full name |
| email | String | Unique email address |
| password | String | Password |
| role | String | `user` or `provider` |

### Service
| Field | Type | Description |
|---|---|---|
| id | Integer | Primary key |
| title | String | Service title |
| description | String | What the service offers |
| role | String | Skill type (Electrician, Plumber…) |
| location | String | Area or city |
| contact | String | Phone / WhatsApp number |
| user_id | Integer | FK → User who posted it |

### Booking
| Field | Type | Description |
|---|---|---|
| id | Integer | Primary key |
| service_id | Integer | FK → Service |
| name | String | Booker's name |
| phone | String | Booker's phone number |
| message | String | Request details |
| created_at | DateTime | Auto-generated timestamp |

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Health check |
| POST | `/signup` | Register a new user |
| POST | `/login` | Login — returns user info and role |
| GET | `/services/` | Get all service listings |
| POST | `/services/` | Create a new listing (provider) |
| PUT | `/services/{id}` | Update an existing listing |
| DELETE | `/services/{id}` | Delete a listing |
| POST | `/bookings/` | Submit a booking request |
| GET | `/bookings/` | Get all bookings (latest first) |

---

## How It Works

```
Provider signs up → posts service (title, role, location, contact)
         ↓
User searches services — real-time filter by title / role / location
         ↓
User clicks Book → inline form expands → fills name, phone, message
         ↓
Booking saved to database with auto timestamp
         ↓
Provider contacted directly via Call / WhatsApp button
```

## Setup & Run

### Prerequisites
- Python 3.10+
- Node.js 16+ and npm

---

### Backend

```bash
# 1. Clone the repo
git clone https://github.com/VishnuMohanp-10/Localskill---Service-Marketplace-Platform.git
cd Localskill---Service-Marketplace-Platform/backend

# 2. Create and activate virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Start the server
uvicorn main:app --reload
```

✅ Backend running at: `http://localhost:8000`  
📄 Interactive API docs: `http://localhost:8000/docs`

---

### Frontend

```bash
# Open a second terminal
cd Localskill---Service-Marketplace-Platform/frontend

npm install
npm start
```

✅ Frontend running at: `http://localhost:3000`

---

### Both Terminals Side by Side

| Terminal | Command | URL |
|---|---|---|
| Terminal 1 — Backend | `uvicorn main:app --reload` | http://localhost:8000 |
| Terminal 2 — Frontend | `npm start` | http://localhost:3000 |

Open **http://localhost:3000** in your browser — that's the full app.

## Example API Usage

### Register
```http
POST /signup
Content-Type: application/json

{
  "name": "Vishnu",
  "email": "vishnu@example.com",
  "password": "pass123",
  "role": "provider"
}
```

### Login
```http
POST /login
Content-Type: application/json

{
  "email": "vishnu@example.com",
  "password": "pass123"
}
```

### Create a Service Listing
```http
POST /services/
Content-Type: application/json

{
  "title": "Home Electrical Repair",
  "description": "Fix wiring, switches, fans and more",
  "role": "Electrician",
  "location": "Kochi, Kerala",
  "contact": "9876543210",
  "user_id": 1
}
```

### Book a Service
```http
POST /bookings/
Content-Type: application/json

{
  "service_id": 1,
  "name": "Rahul",
  "phone": "9876543210",
  "message": "Need electrical repair this weekend"
}
```
